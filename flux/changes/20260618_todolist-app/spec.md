# Specification: Todolist 桌面摆件应用

## 1. 概述

本文档定义了 Todolist 桌面摆件应用的功能规范和技术规范。该应用是一个运行在 Windows 和 Mac 平台的桌面待办事项管理工具，具有桌面摆件特性（悬浮窗口、主题自定义、动画效果等），并支持使用 GitHub 仓库进行云端备份和多设备同步。

## 2. 功能需求

### 2.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 待办事项 CRUD | 创建、查看、编辑、删除、标记完成待办事项 | P0 |
| 优先级管理 | 支持 4 级优先级（低/中/高/紧急 | P0 |
| 分类标签 | 自定义分类标签，支持颜色标识 | P1 |
| 截止日期 | 设置截止日期，支持提醒通知 | P1 |
| GitHub 同步 | 使用 GitHub 仓库备份数据，支持多设备同步 | P0 |
| 数据统计 | 完成率、分类占比、趋势图表 | P1 |
| 主题系统 | 浅色/深色主题切换，支持自定义主题 | P1 |
| 桌面摆件特性 | 窗口置顶、无边框、可拖拽、透明度调节 | P0 |
| 动画效果 | 平滑过渡动画、交互动效 | P2 |
| 系统托盘 | 托盘图标、通知提醒 | P1 |

### 2.2 用户角色

- 单用户应用，无需多设备同步

### 2.3 非功能需求

| 需求 | 描述 |
|------|------|
| 性能 | 启动时间 < 2s，UI 动画 60fps，数据查询 < 50ms |
| 可用性 | 离线可用，网络恢复后自动同步 |
| 安全性 | GitHub Token 加密存储，数据传输使用 HTTPS |
| 兼容性 | Windows 10+, macOS 11+ |
| 可访问性 | 支持键盘快捷键，高对比度主题 |

## 3. 技术规范

### 3.1 技术栈

- **语言**: C++17, QML
- **框架**: Qt 6.5+
- **UI**: Qt Quick (QML)
- **数据库**: SQLite 3
- **GitHub 集成**: libgit2
- **JSON**: nlohmann/json
- **构建系统**: CMake 3.20+
- **测试框架**: Qt Test

### 3.2 系统架构

采用三层架构：表现层（QML）→ 业务逻辑层（C++）→ 数据访问层（C++）
